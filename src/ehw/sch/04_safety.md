# 安全回路（ESTOP / 安全触边 / HSM region=2）

> 设计原则：急停与安全触边**硬件直切动力**，独立于软件；与 HSM 顶层状态机「区域2 安全监控区（region=2，最高优先级）」对应，
> 符合 ISO 3691-4（工业车辆安全）。软件仅负责**监测并上报**安全状态（is_safe / current_state region=2）。

## 1. 硬件直切动力链（独立于 MCU）

```
[ESTOP 急停按钮 NC] ─┐
[安全触边 NC]        ─┤ 串联
[MCU 安全心跳(使能)]  ─┘──► U9(光耦 TLP281) ──► Q11(AO3400 使能 FET) ──► K1(安全继电器/SSR)
                                                                              │
                                                              K1 触点串入 伺服 48V 使能(STOPA/STOPB)
```

- 任一条断开（急停按下 / 触边受压 / MCU 心跳丢失）→ 光耦 U9 截止 → Q11 关断 → K1 释放 →
  伺服 D_00/D_01 的使能链（STOPA/STOPB）断电 → **动力切除**，无需 MCU 软件参与。
- K1 触点容量需覆盖伺服驱动使能线圈（典型 24V/≤100mA 控制回路，或直切 48V 动力接触器线圈）。
- ESTOP 与安全触边采用 **NC（常闭）** 触点：断线即视为故障→安全（fail-safe）。

## 2. MCU 安全心跳（watchdog-driven enable）

- MCU 固件正常运行时，由 GPIO（PB2）周期翻转「安全心跳」信号驱动 U9 输入侧。
- 若固件挂死：IWDG 超时复位（≈1s）**或** 心跳停摆 → U9 截止 → K1 释放 → 自动进入安全态。
- 这是 HSM「区域2 安全监控区（region=2）」的**硬件兜底**：即使软件（含区域0/1/2）崩溃，动力仍被硬件切除。

## 3. 安全状态监测与上报（软件侧）

| 信号 | MCU 引脚 | 含义 |
|------|----------|------|
| SAFE_STATE | PC3（ADC/数字输入） | 读取 K1 实际状态（反馈确认动力已切/未切） |
| ESTOP_IN | PC2（数字输入，经光耦） | 急停按钮实际状态 |
| EDGE_IN | PC13（数字输入，经光耦） | 安全触边状态 |

- 上述经光耦隔离输入，避免现场地噪声误触发；电平变化触发 EXTI 中断 → 置 `is_safe=false`、
  并发 `EV_OBSTACLE_DETECTED`/安全事件（与 CFW 事件总线对齐）。
- HMI 经 `telemetry.is_safe` / `current_state`（region=2, ESTOP 全局捕获）展示与记录。

## 4. 与 HSM 正交区域映射

| HSM 区域 | 硬件对应 |
|----------|----------|
| 区域0 主业务 | MCU 正常运行、心跳输出 |
| 区域1 能源管理 | 由 CFW 监测 battery_soc（ADC），硬件无关 |
| 区域2 安全监控（最高优先级，ADR-006 四级容错硬件兜底层） | 本回路：ESTOP/触边/心跳 → 硬件直切；全局捕获 ESTOP |

> 安全回路为「正交区域」中的**硬件仲裁层**：优先级高于所有软件状态，符合 ADR-003 region=2 最高仲裁。
